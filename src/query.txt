SQL Editor에서 다음 코드를 실행하여 기존 트리거를 삭제하고, 함수와 트리거를 안전하게 재생성할 수 있습니다.

1단계: 기존 트리거 및 함수 삭제
먼저 기존에 존재할 수 있는 트리거와 함수를 삭제합니다.

-- 1. 기존 트리거 삭제 시도 (public.user 테이블에 걸려있을 수도 있으므로 둘 다 시도)
-- auth.users 테이블의 트리거 삭제
DROP TRIGGER IF EXISTS on_auth_user_created ON auth.users;

-- public.user 테이블에 혹시 걸려있는 트리거가 있다면 삭제 (오류 메시지에 'relation "users"'가 있어 확인용)
-- 테이블 이름이 'user'가 아닌 'users'일 가능성도 있으므로, 정확한 테이블명으로 수정 필요
-- DROP TRIGGER IF EXISTS on_auth_user_created ON public.user; 
-- (만약 public.user 테이블에 트리거를 걸었다면 위의 주석을 풀고 실행)

-- 2. 기존 함수 삭제 시도
DROP FUNCTION IF EXISTS public.handle_new_user();


2단계: 함수 및 트리거 재생성 (새로운 코드 실행)
이제 오류 없이 새로운 함수와 트리거를 생성합니다. CREATE OR REPLACE FUNCTION을 사용하면 기존 함수가 있어도 덮어쓰기 때문에 더 안전합니다.

-- 1. 새로운 사용자 처리를 위한 PostgreSQL 함수 생성 (CREATE OR REPLACE로 안전하게)
create or replace function public.handle_new_user()
returns trigger 
language plpgsql
security definer set search_path = public
as $$
begin
  -- public.user 테이블에 auth.users의 id와 email을 삽입합니다.
  -- service_agreed와 privacy_agreed는 NULL로 남겨두고, marketing_agreed는 DEFAULT FALSE가 적용됩니다.
  insert into public.user (id, email, service_agreed, privacy_agreed)
  values (new.id, new.email, NULL, NULL);
  
  return new;
end;
$$;

-- 2. auth.users 테이블에 insert 이벤트가 발생할 때마다 함수를 실행하도록 트리거 생성
create trigger on_auth_user_created
  after insert on auth.users -- auth.users에 행이 삽입된 '후'에 실행
  for each row execute function public.handle_new_user();
